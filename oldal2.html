Git Parancsok

git config --global user.name "Github-on megadott név" és git config --global user.email "Github-on regisztrált emailcím"

Ezzekel a parancsokkal lehet megadni a fiok infórmációt hogy fell tudd tölteni a git-be amelyikbe akarod.

git init

Ezzel a parancsal lehet inicializálni a mappát, hogy majd fel lehessen tölteni GitHubra.

git remote add origin repository-link (github-ból code-ból elérhető a közvetlen link)

Ezzel pedig a Repository-it tudod megadni amelyikbe fel fogod tölteni a változtatásokat.

git remote -v

Meg lehet nézni, hogy jó-e a kapcsolat a kliens és a repo között.

git add file-name

Amikor csinálsz egy változtatást vagy egy fájlt akkor ezzel a parancsal lehet betenni a staging-area-ba.

git add .

Ez akkor hasznos amikor mindent be akarsz rakni a mappából.

git status

Meglehet nézni a fájlokat amik benne vannak a staging-area-ba.

git commit -m "Kötelező megjegyzést tenni"

A staging-area-ból a repóba való töltéshez ezt a parancsot kell használni.

git log

Eddigi verzió megtekintése ami valahogy így fog kinézni
commit a661d9249509f8aa8f81f6f58129d57eb36b81d6 Author: Gergo Roth roth@dcs.uni-pannon.hu Date: Wed Jul 6 18:36:01 2016 +0200 My first commit.

git push origin master

Ezzel lehet feltölteni véglegesen a GitHubra.

Példa

git init
git config --global user.name "Github-os név"
git config --global user.email "Github-on megadott emailcím"
git remote add origin repository-link
git remote -v (ezt ki lehet hagyni, de jó ötlet biztonság kedvéért
git add .
git status (ez is ki hagyható)
git commit -m "Kötelező megjegyzés"
git log (ugyanúgy kihagyható)
git remote -v (megint kihagyható) git push origin master
